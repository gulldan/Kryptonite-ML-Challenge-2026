# Карточка модели: Kryptonite Speaker Retrieval

Документ описывает текущее challenge-решение. Это не паспорт промышленной
биометрической системы, а краткая карточка модели для проверки решения жюри.

## Статус

Текущий кандидат:

- идентификатор запуска: `MS41_ms32_classaware_c4_weak_20260415T0530Z`
- score на открытой части теста: `0.7473`
- основная модель: CAM++ с официальным модулем аудиопризнаков 3D-Speaker/ModelScope
- выход: `submission.csv` с 10 ближайшими соседями для каждой строки тестового CSV

Score на открытой части теста используется только как внешний ориентир. Финальная оценка
считается организаторами на закрытой private-выборке.

## Для чего модель используется

Подход подходит для:

- challenge-задачи speaker retrieval;
- сравнения аудиофайлов внутри одной тестовой выборки;
- построения `submission.csv` в формате организаторов;
- исследовательских экспериментов с моделями, псевдометками и графовой постобработкой.

Подход не надо трактовать как:

- готовую промышленную систему биометрической аутентификации;
- юридическое или forensic-решение;
- систему с доказанной fairness-оценкой по демографическим группам.

## Короткое описание пайплайна

1. Аудио читается и приводится к `16 kHz`, mono.
2. Признаки строятся через официальный CAM++ frontend.
3. CAM++ encoder строит числовой вектор голоса для каждого файла.
4. По векторам голосов ищутся ближайшие соседи.
5. Графовая постобработка усиливает взаимные устойчивые связи и снижает число чрезмерно
   популярных соседей.
6. Слабая class-aware поправка добавляет небольшой бонус похожим парам, но не делает
   жёсткого назначения класса.
7. Валидатор проверяет итоговый `submission.csv`.

## Данные и обучение

| Источник | Роль |
| --- | --- |
| Данные организаторов `train.csv` + `train/` | Основное supervised-обучение и fine-tuning. |
| `test_public.csv` + `test_public/` | Тестовый домен без лейблов; использовался для инференса, анализа графа соседей и дообучения по псевдометкам без доступа к скрытым ответам. |
| Public-псевдометки | Кластеры похожих public-файлов, отфильтрованные по надёжности. |
| Аугментации | Шумы, реверберация, codec/distortion, speed perturbation, тишина и изменение громкости для устойчивости к каналу. |
| Внешние датасеты | Использовались как исследовательские ветки; не стали финальной основой. |

Среда обучения: финальные ветки обучались на GPU внутри Docker-образа
`nvcr.io/nvidia/pytorch:26.03-py3`.

## Основные риски

- Public и private могут отличаться. Поэтому score на открытой части не гарантирует score
  на закрытой части.
- Локальные proxy-метрики иногда были оптимистичны; это зафиксировано в истории
  экспериментов.
- Финальное качество зависит от корректного набора весов и больших артефактов в
  `artifacts/`.
- Первый полный прогон из аудио ограничен скоростью извлечения аудиопризнаков; TensorRT
  ускоряет encoder, но не отменяет стоимость этого этапа.
