# Контракт задачи

Этот документ фиксирует, что именно решает репозиторий в рамках Kryptonite ML Challenge
2026 и какой выход считается корректным.

## Тип задачи

Основной режим: speaker retrieval внутри одной тестовой выборки.

Для каждой строки тестового CSV нужно вернуть 10 индексов других строк из того же CSV.
Индекс означает позицию аудиофайла в тестовом списке. Self-match запрещён: файл не может
быть собственным соседом.

## Вход

Ожидается CSV со столбцом:

- `filepath` - путь к аудиофайлу относительно корня датасета.

Для public организаторский файл лежит здесь:

```text
datasets/Для участников/test_public.csv
```

Для private используется такой же формат.

## Выход

Итоговый файл должен называться `submission.csv` и содержать два столбца:

- `filepath` - тот же порядок и те же значения, что во входном CSV;
- `neighbours` - строка из 10 целых индексов через запятую.

В столбце `neighbours` индексы записываются одной строкой через запятую.

## Правила корректности

Валидатор проверяет:

- число строк совпадает с шаблонным CSV;
- порядок `filepath` не изменён;
- в каждой строке ровно 10 соседей;
- нет пустых значений;
- нет повторов индексов;
- нет совпадения файла с самим собой;
- все индексы находятся внутри диапазона тестовой выборки.

Команда проверки:

```bash
uv run python scripts/validate_submission.py \
  --template-csv datasets/Для\ участников/test_public.csv \
  --submission-csv submission.csv \
  --output-json artifacts/release/submission_validation.json
```

## Метрика

Организаторы считают `Precision@10` по скрытым speaker labels. Если среди 10 соседей
одной строки 7 файлов того же диктора, вклад этой строки равен `0.7`. Итоговая оценка -
среднее по строкам.

Public labels скрыты, поэтому score на открытой части теста нельзя честно пересчитать
локально. Private score считается только организаторами.

## Текущий рабочий пайплайн

1. Прочитать тестовый CSV и аудио.
2. Построить CAM++ audio features.
3. Получить векторы голосов.
4. Найти top-200 ближайших кандидатов.
5. Применить графовую постобработку и слабую class-aware поправку.
6. Записать 10 лучших соседей в `submission.csv`.
7. Прогнать валидатор формата.

Практические команды запуска описаны в [../README.md](../README.md).
