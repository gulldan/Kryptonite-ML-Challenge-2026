# Руководство для жюри (быстрое ознакомление)

Эта секция для тех, кто оценивает проект и хочет быстро понять, что уже работает,  
не вчитываясь во все технические заметки.

## Что уже работает «из коробки»

- Запуск API + веб-дэмо (`/demo`) одной командой через `docker compose up --build`.
- Проверка состояния сервиса через `GET /health`.
- Просмотр решения на одном аудиопримере через UI demo.
- Отдельные workflow для аудионастройки, обучения и валидации находятся в `scripts/`.

## Что важно для оценки в 5 минут

1. Запустите репозиторий по `README.md`.
2. Откройте `/demo` и убедитесь, что сравнение аудио даёт числовой score.
3. Откройте `/health` и убедитесь, что сервис стартует в корректном режиме.
4. Убедитесь, что есть reproducible-подход:
   - версии зависимостей в `uv.lock`,
   - конфиги в `configs/`,
   - скрипты запуска и отчётов в `scripts/`.

## Что за архитектура (без внутреннего шума)

- `src/kryptonite/` — ядро (processing/eda/training/inference interfaces).
- `apps/api/` — REST entrypoint на FastAPI.
- `apps/web/` — React + Vite демо UI.
- `deployment/` — инструкции и docker-файлы для развёртывания.
- `docs/` — только для углубления, если нужно.
