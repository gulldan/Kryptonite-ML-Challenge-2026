# Контракт аудио-пайплайна

## Цель

Это единая справка по аудио-контракту репозитория. Она нужна, чтобы не открывать
несколько исторических заметок ради нормализации, VAD, Fbank и правил нарезки.

## Основной контракт

- частота дискретизации: `16 kHz`
- каналы: `mono`
- формат выхода: `PCM16 WAV`
- признаки: `80`-bin log-Mel / Fbank
- preprocessing явный и управляется конфигами
- обучение, оценка и runtime используют одно семейство loader/frontend

## Порядок обработки

1. декодирование, resample, сведение каналов
2. опциональная ограниченная нормализация громкости
3. опциональный VAD-trimming только по краям
4. нарезка utterance на фрагменты
5. извлечение Fbank
6. pooling фрагментов и последующий embedding/scoring

## Runtime-параметры по умолчанию

- нормализация громкости поддерживается, по умолчанию выключена
- режим VAD по умолчанию: `none`
- окно training crop: `1-4 s`
- окно eval/demo: фрагменты по `4 s` с overlap по умолчанию
- pooling явный, а не скрытое поведение downstream-кода

## Устойчивость к искажениям

Подробности по corruption banks, silence augmentation и scheduler policy убраны из
видимой документации для жюри. Текущий контракт: изменения, которые реально меняют
waveform, границы сегментов, frontend tensors или сопоставимость evaluation, должны
жить в конфиге и воспроизводимом коде, а не только в заметке.

## Правило использования

Если поведение важно для финального решения, оно должно быть в `configs/`, `src/` или
`scripts/` и упомянуто в [challenge-experiment-history.md](../challenge-experiment-history.md).
