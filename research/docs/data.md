# Данные

Этот документ коротко объясняет, какие данные использует решение и где они должны лежать.

## Основные данные задачи

Данные организаторов не хранятся в git. Локально они ожидаются в папке:

```text
datasets/Для участников/
```

Ожидаемая структура:

```text
datasets/Для участников/
├── train.csv
├── test_public.csv
├── baseline.onnx
├── train/
└── test_public/
```

Для private-запуска организаторы должны дать CSV такого же формата и папку с аудиофайлами.

## Что хранится в git

В git хранятся:

- код чтения и проверки данных;
- схемы manifests;
- конфиги;
- документация;
- тесты.

В git не хранятся:

- аудиоданные;
- веса моделей;
- embeddings;
- кеши признаков;
- файлы отправки, если они большие или сгенерированы локально.

## Где код данных

| Путь | Назначение |
| --- | --- |
| `src/kryptonite/data/` | Чтение аудио, манифесты, валидация, VAD, нормализация. |
| `src/kryptonite/features/` | Аудиопризнаки, CAM++ frontend, fbank, нарезка аудио, кеши. |
| `src/kryptonite/eda/` | Анализ датасета, сравнение train/public, проверка submission. |
| `scripts/build_participant_training_manifests.py` | Сборка manifests для обучения из данных участников. |
| `scripts/run_eda_profile.py` | Базовый EDA-профиль данных. |

## Практические правила

- Не коммитить `datasets/`.
- Не полагаться на случайные локальные пути. Пути должны задаваться через конфиг или CLI.
- Важные сгенерированные результаты записывать в `artifacts/` и в историю экспериментов.
- Если private CSV отличается от public CSV только содержимым строк, команды из runbook
  должны работать без правки кода.
